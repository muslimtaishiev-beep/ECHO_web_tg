1. Введение
1.1. Цель проекта
Разработать веб-платформу для предоставления анонимной психологической помощи подросткам. Платформа обеспечивает конфиденциальность пользователей, интеграцию чата в реальном времени, регистрацию пользователей и волонтёров, а также административный функционал с ролевой моделью доступа (RBAC).

1.2. Обоснование выбора NestJS вместо Spring Boot
Оригинальное ТЗ предполагало Java/Spring Boot. После анализа готового фронтенда (React + Vite, TypeScript/JavaScript) принято решение перейти на NestJS по следующим причинам:

Единая экосистема — TypeScript используется и на фронте, и на бэке. Типы, DTO и интерфейсы можно шарить через общие пакеты.
Нативная поддержка WebSocket — NestJS Gateway + Socket.io из коробки, без дополнительной настройки Spring WebSocket.
Скорость разработки — npm-экосистема, знакомые инструменты, единый репозиторий (monorepo).
Архитектурная схожесть — NestJS использует те же паттерны: модули, DI-контейнер, декораторы, guards (аналог Spring Security Filters).
Производительность — Node.js с кластеризацией и Socket.io Redis adapter покрывает 1000+ одновременных пользователей.

1.4. Общие требования
Анонимность: все пользовательские данные (включая чаты) шифруются на сервере (AES-256-GCM). Доступ к расшифрованным данным только через защищённую админ-панель с RBAC. IP-адреса маскируются в логах.
Дизайн: полное соответствие готовому макету (Figma/PNG). Сайт адаптивен (responsive) для мобильных устройств.
Безопасность: соответствие GDPR, защита от SQL-инъекций (TypeORM parameterized queries), XSS (Helmet CSP), CSRF (@nestjs/csrf). Регулярное сканирование уязвимостей.
Производительность: поддержка до 1000 одновременных пользователей, время отклика API < 200 мс, Socket.io с Redis adapter для горизонтального масштабирования.

1.5. Этапы разработки
Анализ и проектирование — 2 недели. Финализация ТЗ, ERD-схема, Swagger-контракт, дизайн-макет.
Разработка — 8–10 недель. Фронтенд и бэкенд параллельно, с еженедельной синхронизацией через API-контракт.
Интеграция и тестирование — 3 недели. E2E, нагрузочные тесты (JMeter), security audit.
Деплой и мониторинг — 1 неделя. Kubernetes-деплой, настройка Prometheus/Grafana, Go-live.

1.6. Структура команды
Frontend-разработчик: React + Vite, Socket.io-client, адаптация к REST API.
Backend-разработчик: NestJS, TypeORM, Socket.io Gateway, безопасность.
DevOps-инженер: Docker, Kubernetes, GitHub Actions, Nginx, мониторинг.

2. Фронтенд-разработка
2.1. Обязанности
Реализовать UI/UX на основе предоставленного дизайн-макета (все страницы, компоненты, стили).
Обеспечить анонимность: нет сбора личных данных без согласия; формы регистрации без обязательных ПИИ.
Интеграция с бэкендом через REST API (Axios) и WebSocket (Socket.io-client).
Хранение JWT Access/Refresh токенов в httpOnly cookie (не localStorage).

2.2. Ключевые страницы и функции
Главная страница
Баннер с описанием платформы, кнопка анонимного входа в чат.
FAQ-секция. Кнопка «Написать волонтёру» открывает выбор темы.
Регистрация / Логин
Пользователи (подростки): анонимная регистрация (никнейм обязателен, email — опционально). Без реальных ФИО.
Волонтёры / персонал: расширенная форма, верификация через админ-панель.
Сброс пароля по email (анонимный flow, без раскрытия личности).
Чат-модуль
Real-time чат через Socket.io-client. Подключение к анонимной комнате по теме.
Функции: отправка текста, эмодзи; отображение статуса онлайн/оффлайн волонтёра.
История чатов — только для текущей сессии (зашифрована на клиенте через SubtleCrypto).
Пользовательский профиль
Минимальный: изменение ника, список завершённых чатов (без содержимого по умолчанию).
Админ-панель
Дашборд: статистика (количество пользователей, чатов, волонтёров) — анонимно.
Управление пользователями: поиск по ID/нику, блокировка аккаунтов, просмотр истории (с расшифровкой через бэкенд).
Управление волонтёрами: добавление/удаление, назначение ролей.
Журнал аудита: действия администраторов с timestamp.
Экспорт данных: CSV/JSON с шифрованием (только для суперадмина).

2.3. Технические требования
Фреймворк: React 18 + Vite (текущий стек, без изменений).
Стили: CSS/SCSS, следование макету (цвета, шрифты, отступы). Tailwind CSS — опционально.
Состояние: Zustand или React Query для серверного состояния (кеширование API).
WebSocket: Socket.io-client 4.x, переподключение при разрыве, анонимный handshake.
Тестирование: Jest + React Testing Library (unit), Cypress (E2E).
Доступность: WCAG 2.1 AA — альт-тексты, клавиатурная навигация, контрастность.

2.4. Доставляемые артефакты
Исходный код в Git (ветки: main, develop, feature/*).

3.5. WebSocket Gateway (Socket.io)
Namespace: /chat
Events (client → server): join_room, send_message, leave_room, typing.
Events (server → client): message, user_joined, user_left, volunteer_assigned, error.
Аутентификация: JWT в handshake query (?token=...), проверка через JwtAuthGuard.
Redis Adapter: @socket.io/redis-adapter для горизонтального масштабирования на K8s.
Rate limiting: 100 сообщений/мин на пользователя через ThrottlerGuard.

3.6. Безопасность
AES-256-GCM: все PII-поля (nickname, email, content) шифруются в TypeORM @BeforeInsert/@BeforeUpdate трансформерах.
Ключи шифрования: хранятся в HashiCorp Vault, загружаются при старте через @nestjs/config + Vault SDK.
JWT: Access Token (15 мин), Refresh Token (7 дней, httpOnly cookie). Ротация при каждом обновлении.
Helmet: настройка CSP, X-Frame-Options, HSTS.
RBAC: кастомный @Roles() декоратор + RolesGuard на всех защищённых маршрутах.
SQL-инъекции: TypeORM использует параметризованные запросы. Raw SQL запрещён.
Логи: IP-адреса маскируются (заменяются на хеш). Аудит-лог в AdminLog entity.

3.7. Технические требования
Node.js 20 LTS, NestJS 10.x, TypeScript 5.x.
TypeORM 0.3.x + PostgreSQL 16. Только type-safe запросы.
@nestjs/swagger — OpenAPI 3.0 документация, доступна на /api/docs.
Winston для структурированного логирования (JSON-формат, без PII).
Тестирование: Jest (unit, coverage > 80%), Supertest (E2E), JMeter (нагрузка).

4.4. Мониторинг и безопасность
Prometheus: сбор метрик из NestJS (@willsoto/nestjs-prometheus), Nginx, PostgreSQL exporter.
Grafana: дашборды — CPU/RAM, RPS, latency p95, активные WebSocket-соединения, ошибки 4xx/5xx.
Winston → stdout → Fluentd → Elasticsearch → Kibana (ELK Stack).
Алерты: PagerDuty/Slack при ошибках > 1% или latency > 500 мс.
Бэкапы: ежедневный pg_dump с шифрованием, хранение в S3 (30 дней).
Firewall: открыты только порты 80, 443. PostgreSQL и Redis — только внутри кластера.

4.5. Технические требования
IaC: Terraform для provisioning облака (AWS EKS или GCP GKE).
Секреты: HashiCorp Vault, Vault Agent Sidecar Injector.
Нагрузочное тестирование: JMeter, сценарий 1000 одновременных WebSocket-соединений.
SSL: Let's Encrypt через cert-manager, автообновление.
