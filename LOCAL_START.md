# 🚀 Локальный старт проекта (ECHO Local)

Эти команды позволяют запустить проект полностью локально без использования Docker. Для базы данных используется **SQLite**.

## 🛠 Предварительные требования
*   **Node.js** (рекомендуется v20+)
*   **npm**

## ⚡️ Быстрый старт
```bash
# 1. Установка зависимостей (если еще не сделано)
npm run install:all

# 2. Однокликовый запуск (настроит окружение и запустит сервер + клиент)
npm run local:start
```

## 📋 Что делает `npm run local:start`?
1.  **Настраивает окружение**: Запускает скрипт `setup-local-db.js`, который:
    *   Создает файл `.env` (если его нет) или делает бэкап существующего Railway конфига в `.env.railway`.
    *   Устанавливает `DATABASE_URL` на локальный файл SQLite (`dev.db`).
    *   Генерирует локальную схему Prisma (`schema.local.prisma`) с провайдером `sqlite`.
2.  **Синхронизирует БД**: Создает таблицы в локальном файле `dev.db`.
3.  **Запускает проект**: Запускает NestJS (Backend) и Vite (Frontend) одновременно.

## 🔍 Проверка
*   **Backend**: [http://localhost:3000](http://localhost:3000)
*   **Frontend**: [http://localhost:5173](http://localhost:5173) (или 5174, проверьте консоль)

## 🔄 Переключение обратно на Railway
Для деплоя на Railway проект по-прежнему использует основной `schema.prisma` и переменные окружения, заданные на платформе. Локальные изменения в `.env` игнорируются при деплое (через `.gitignore`).
